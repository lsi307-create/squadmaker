# Compound Learning Store

This directory is the project memory for solved Squadmaker problems and decisions.

Use it to keep future work faster:

- Record why a behavior exists, not only what changed.
- Prefer one focused note per feature, bug, or real-use lesson.
- Link to related tests, source files, design docs, and deployment commits.
- Keep notes practical enough for the next implementation session.

Recommended filename:

`docs/solutions/<category>/<short-topic>-YYYY-MM-DD.md`

Current categories:

- `product/`: product behavior, operating rules, manager workflow
- `frontend/`: UI layout, mobile interaction, visual design
- `deployment/`: GitHub Pages or future Cloudflare Pages deployment notes
- `testing/`: test patterns and real-use test findings
