# Squadmaker V2 관리자 가이드

## 목적

Squadmaker V2는 경기 당일 운영자가 모바일에서 출석 인원, 팀 배정, 포메이션, 쿼터별 출전 기록을 빠르게 관리하기 위한 앱이다. 핵심 목표는 복잡한 기록 앱이 아니라, 현장에서 팀을 공정하고 빠르게 나누는 것이다.

## 기본 화면 구조

앱은 모바일 하단 탭 3개로 구성한다.

- 참여인원: 전체 명단에서 오늘 온 사람을 선택하고, 출석 순서와 쿼터 수를 확인한다.
- A팀: A팀 선수 배치와 포메이션을 확인하고 조정한다.
- B팀: B팀 선수 배치와 포메이션을 확인하고 조정한다.

## 명단 관리

처음 앱에는 기본 선수 명단이 들어 있다. 선수 정보는 이름만 사용한다.

관리자는 필요할 때 명단을 붙여넣거나 수정할 수 있다. 출석률, 출석순위, 능력치, 메모 같은 정보는 첫 버전에서는 넣지 않는다. 앱이 복잡해지는 것을 막기 위해서다.

## 당일 출석 처리

참여인원 탭에서 선수를 탭하면 오늘 출석으로 처리된다.

출석 처리된 선수는 자동으로 도착 순서가 부여된다. 이 순서는 사람이 중간에 빠지거나 복귀할 때 당일 운영 흐름을 이해하는 기준으로 사용한다.

선수 상태는 단순하게 유지한다.

- 출석: 현재 배정 대상이다.
- 잠시 제외: 잠깐 쉬거나 부상, 전화 등으로 다음 배정에서 빠진다.
- 복귀: 다시 배정 대상이 된다.
- 오늘 이탈: 오늘 경기에서 더 이상 배정하지 않는다.

출석한 선수를 다시 탭하면 하단 메뉴가 열리고, 관리자는 상태를 바꿀 수 있다.

## 팀 배정

팀 배정은 자동 배정을 먼저 사용한다. 이후 관리자가 필요하면 선수끼리 직접 교체한다.

기본 팀 구성은 11명 기준이다. 433, 4231, 532 포메이션을 기준으로 A팀과 B팀을 구성한다.

출석 인원이 부족하면 일부 포지션은 비워둘 수 있다. 출석 인원이 많으면 남는 인원은 대기자로 분리된다.

## 키퍼 규칙

키퍼 운영은 Squadmaker V2의 중요한 공정성 규칙이다.

- 앱은 각 팀의 키퍼를 자동 추천한다.
- 관리자는 추천된 키퍼를 직접 바꿀 수 있다.
- 부상이나 컨디션 문제로 필드는 어렵지만 키퍼는 가능한 사람이 있을 수 있으므로, 수동 변경 권한을 항상 둔다.
- 키퍼로 뛴 쿼터는 필드 출전 쿼터 수에 포함하지 않는다.
- 직전 쿼터에 키퍼였던 선수는 다음 쿼터에 필드 플레이어로 들어가는 것을 우선 보장한다.
- 직전 쿼터에 키퍼였던 선수는 다음 키퍼 후보에서는 후순위로 밀린다. 키퍼를 고정하지 않기 위한 규칙이다.

## 선호 포지션

선수 상태 메뉴에서 선호 포지션을 선택할 수 있다.

- 기본값은 랜덤이다.
- 선택지는 수비, 미드, 공격, GK다.
- 선호 포지션은 강제 배정이 아니라 자동 배정 추천에 반영되는 참고값이다.
- 관리자는 자동 배정 후 포메이션 화면에서 선수끼리 탭해 직접 자리를 바꿀 수 있다.

## 쿼터 종료

쿼터 종료 시 앱은 현재 A팀과 B팀의 필드 플레이어 출전 수를 올린다. 키퍼는 별도의 키퍼 출전 수로 기록하고, 필드 출전 수에는 더하지 않는다.

다음 쿼터 자동 배정에서는 다음 조건을 우선 고려한다.

- 현재 출석 상태인 선수만 배정한다.
- 잠시 제외 또는 오늘 이탈 선수는 배정하지 않는다.
- 직전 쿼터 키퍼였던 선수는 필드 출전을 우선 보장한다.
- 직전 쿼터 키퍼였던 선수는 다음 키퍼 추천에서 후순위가 된다.
- 필드 출전 수가 적은 선수를 우선 고려한다.
- 선호 포지션이 있으면 현재 포메이션의 수비, 미드, 공격, GK 슬롯에 가능한 범위에서 반영한다.
- 관리자가 마지막에 수동으로 조정할 수 있다.

## 첫 배포 범위

첫 배포는 모바일 웹앱으로 만든다. GitHub Pages 또는 Cloudflare Pages에 배포할 수 있어야 한다.

Android와 iOS 앱 출시는 다음 단계에서 보완한다. 첫 버전에서는 앱 출시보다 경기 당일 실제 사용이 가능한 안정적인 웹앱을 우선한다.
