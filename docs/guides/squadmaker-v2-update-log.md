## 주요 업데이트

### 1. 모바일 앱 구조

- 하단 탭 3개로 구성했다.
- 탭은 `참여인원`, `A팀`, `B팀`이다.
- PC보다 모바일 사용성을 우선한다.
- 첫 화면은 랜딩 페이지가 아니라 바로 경기 당일 운영 화면이다.

### 2. 참여인원

- 기본 명단에서 이름을 탭하면 당일 출석으로 처리된다.
- 출석 순서가 자동으로 기록된다.
- 출석한 선수를 다시 탭하면 상태/선호 포지션을 바꿀 수 있다.
- 검색 기능을 추가했다.
- `김용삼1`은 `김용삼`으로 수정했다.
- 중복 항목 `김용삼2`는 삭제했다.

### 3. 자동 배정

- `추천 배정`을 누르면 A/B 라인업 보드로 인원을 나눈다.
- A/B는 고정 팀이 아니라 현재 쿼터 라인업이다.
- 다음 쿼터 추천은 기존 팀을 유지하는 개념이 아니라 뛴 사람과 덜 뛴 사람을 고려해 새로 짜는 개념이다.
- 필드 출전 수가 낮은 사람이 다음 추천에서 우선된다.

### 4. 포메이션

- 각 팀은 `433`, `4231`, `532` 포메이션을 선택할 수 있다.
- 선발 선수는 포메이션 위치에 강제로 배치된다.
- 포지션 칩을 탭해 선수끼리 위치를 바꿀 수 있다.
- 선호 포지션은 자동 배정에 참고되지만 강제는 아니다.

### 5. 선발과 대기

- 팀 화면에는 `선발 명단`과 `대기 명단`이 함께 표시된다.
- 선발 포지션 선수를 탭한 뒤 대기 선수를 탭하면 교체된다.
- 대기 선수를 먼저 탭한 뒤 포지션 선수를 탭해도 교체된다.
- 대기 명단은 경기 중 사람이 빠지거나 들어올 때 운영자가 직접 조정하기 위한 장치다.

### 6. GK 규칙

- GK는 고정 선수가 아니다.
- GK로 뛴 쿼터는 필드 출전 수에 포함하지 않는다.
- 직전 쿼터 GK였던 사람은 다음 쿼터 필드 출전을 우선 추천받는다.
- 직전 쿼터 GK였던 사람은 다음 GK 추천에서는 후순위가 된다.
- 부상이나 컨디션 문제로 GK가 필요한 경우 관리자가 직접 조정할 수 있다.

### 7. 색상 규칙

- A팀 고정 컬러: 노랑
- B팀 고정 컬러: 파랑
- 공격 포지션: 빨강
- 미드필드 포지션: 초록
- 수비 포지션: 파랑
- GK: 노랑

### 8. 배포와 검증

- GitHub Pages에 정적 웹앱으로 배포했다.
- 마지막 검증에서 테스트 27개가 통과했다.
- 프로덕션 빌드가 통과했다.
- 모바일 화면 `390x844` 기준으로 공개 배포본을 확인했다.
- 공개 배포본에서 검색, 자동 배정, 선발-대기 교체를 확인했다.

## 다음 실사용 테스트에서 볼 것

- 출석 체크가 실제 현장에서 충분히 빠른지.
- 검색이 긴 명단에서 이름 찾기에 충분한지.
- 선발-대기 교체 흐름이 헷갈리지 않는지.
- 자동 배정이 실제 체감상 공정한지.
- GK 추천이 납득 가능한지.
- 포지션 색상이 경기 중 빠르게 이해되는지.
- 쿼터 종료 버튼을 누르는 타이밍이 자연스러운지.
- 경기 중 실수했을 때 되돌리는 기능이 필요한지.
