# Squadmaker V2 설계 문서

## 제품 방향

Squadmaker V2는 모바일 우선 경기 당일 스쿼드 관리 앱이다. 첫 배포는 출석 체크, 임시 제외/복귀, A/B팀 배정, 포메이션 확인, 키퍼 공정성 처리, 쿼터 기록, 정적 배포까지를 목표로 한다.

선택한 방향은 가벼운 접근 B다. 실제 경기 운영에 필요한 관리 기능은 넣되, 계정, 클라우드 동기화, 장기 통계, 앱스토어 출시는 뒤 단계로 미룬다.

## 화면 구조

앱은 하단 3탭으로 구성한다.

- 참여인원: 기본 명단, 오늘 출석, 출석 순서, 선수 상태, 필드 쿼터 수, 키퍼 쿼터 수
- A팀: 포메이션 선택, A팀 배치, 키퍼 지정, 수동 교체
- B팀: 포메이션 선택, B팀 배치, 키퍼 지정, 수동 교체

## 핵심 개념

- 선수: 기본 명단에 있는 사람이다. 첫 버전의 기본 정보는 이름만 저장한다.
- 경기일: 오늘의 출석, 상태, 쿼터, 팀 배정 정보를 포함하는 현재 세션이다.
- 출석 선수: 현재 배정 가능한 선수다.
- 잠시 제외 선수: 출석은 했지만 다음 배정에서 빼야 하는 선수다.
- 오늘 이탈 선수: 오늘 더 이상 배정하지 않는 선수다.
- 필드 쿼터: 골키퍼가 아닌 필드 플레이어로 뛴 쿼터다.
- 키퍼 쿼터: 골키퍼로 뛴 쿼터이며, 필드 쿼터와 따로 기록한다.

## 주요 동작

1. 관리자는 참여인원 탭에서 시작한다.
2. 앱은 기본 선수 명단을 보여준다.
3. 선수를 탭하면 오늘 출석으로 처리하고 출석 순서를 부여한다.
4. 이미 출석한 선수를 탭하면 상태 변경용 하단 메뉴를 연다.
5. 자동 배정은 현재 출석 상태인 선수만 사용한다.
6. 앱은 11명 기준으로 A팀과 B팀을 구성한다.
7. 남는 출석 선수는 대기자로 분리한다.
8. 인원이 부족한 경우 포메이션의 일부 자리는 빈 상태로 둔다.
9. 관리자는 자동 배정 후 선수끼리 수동 교체할 수 있다.
10. 앱은 키퍼를 추천하지만 관리자는 직접 바꿀 수 있다.
11. 쿼터가 종료되면 필드 선수는 필드 쿼터 수가 올라가고 키퍼는 키퍼 쿼터 수가 올라간다.
12. 직전 쿼터 키퍼였던 선수는 다음 쿼터 필드 출전을 우선 보장받는다.

## 포메이션 지원

첫 버전은 각 팀에서 다음 포메이션을 지원한다.

- 433
- 4231
- 532

A팀과 B팀은 각각 독립적으로 포메이션을 볼 수 있어야 한다. 첫 버전에서 포메이션은 배치 화면의 형태를 결정하며, 선수별 선호 포지션이나 능력치 기반 배정은 넣지 않는다.

## 데이터와 저장

첫 버전은 로컬 저장을 사용한다. 최소한 다음 정보가 유지되어야 한다.

- 기본 명단 이름
- 오늘 출석 여부
- 출석 순서
- 선수 상태
- A팀, B팀, 대기자 배정
- 선택한 포메이션
- 필드 쿼터 수
- 키퍼 쿼터 수
- 현재 쿼터

앱에는 현재 경기일을 초기화하는 방법이 있어야 한다. 단, 경기일 초기화가 기본 명단을 삭제하면 안 된다.

## 모바일 UX 원칙

- 경기 중 자주 쓰는 동작은 한 번의 탭에 가깝게 만든다.
- 선수 카드 안에 작은 버튼을 많이 넣지 않고, 상태 변경은 하단 메뉴로 처리한다.
- 참여인원 카드에는 필요한 정보만 간결하게 보여준다.
- 자동화는 운영을 돕는 역할이고, 마지막 결정권은 관리자에게 둔다.
- PC 화면보다 모바일 화면에서 먼저 자연스럽게 보여야 한다.

## 첫 구현 단위

1. 프로젝트 뼈대, 모바일 화면 틀, 하단 3탭
2. 기본 명단, 명단 수정, 로컬 저장
3. 출석 선택, 출석 순서, 하단 상태 메뉴
4. 자동 배정, 대기자 처리, 수동 교체
5. A팀/B팀 포메이션 보드
6. 키퍼 추천, 수동 변경, 쿼터 계산
7. 정적 배포 설정과 모바일 검증

## 보류 범위

- 선수 등급, 출석률, 출석 순위
- 득점, 도움, 상세 경기 기록
- 로그인과 클라우드 백업
- 여러 기기 동시 사용
- Android와 iOS 앱스토어 출시
